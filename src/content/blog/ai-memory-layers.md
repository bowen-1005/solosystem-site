你以為 AI 的記憶問題是「記不住」，於是拼命餵它更多。但真正會害到你的，是它把「會變的」和「不會變的」記在同一個地方，分不出哪一條早就過期了。

如果你已經把 AI 接進工作、餵了一堆背景，卻偶爾發現它用一個你早就改掉的結論在回答你——這篇是寫給你的。我會把 AI 記憶該分的四層講清楚，**每一層都附上我自己在用的寫法，你可以直接複製去改**。不是教學文，是把心法打成你能上手的招式。

## 我差點讓 AI 照著「過期的我」做事

我手上有不只一個事業，一條決策常常擱了幾週才回頭處理。有一次，我針對其中一個事業（就叫它 A）開了一條新支線，給了 AI 新方向。它讀完，很有把握地開始執行——用的卻是我三週前、那條線還沒轉彎時定下的判斷。

把那段對話攤開，大概是這樣：

> **我（三週前）**：A 現階段先守住現金流、暫緩擴張，記住這個方向。
> **我（上週・新對話）**：幫我評估 A 要不要接下這個擴張機會。
> **AI**：依你先前定下的「暫緩擴張」方向，我建議婉拒這個機會……

它沒記錯任何一個字。它忠實地用了「上一個版本的我」。問題是那句「暫緩擴張」是三週前現金吃緊時的判斷，如今情況早就反轉——而我從沒回頭告訴它「那條作廢了」。

那一刻我看懂：問題不在它記不記得，在我把一條**會過期的判斷**，存成了一條**永不過期的事實**，中間沒有任何標籤提醒它，這一條我已經換過了。

## 大多數人的記憶，是一個沒有標籤的倉庫

先講結論：大多數人建 AI 記憶的方式，是「遇到什麼塞什麼」，而這正是它日後記錯的溫床。

專案背景、討論結論、個人偏好、一個臨時的決定——全部堆進同一個地方，沒有分類、沒有標注。它根本不知道這堆東西裡，哪些是鐵打的事實、哪些是這週才成立、下週就會變的判斷。它不知道，所以你也不會知道。

同一個問題，有沒有分層，AI 的回答天差地遠：

| 沒分層（全塞在一起） | 分層後（判斷有標日期） |
|---|---|
| 問「A 該擴張嗎」→ 它抓到舊的「暫緩擴張」當定論，很自信地回你「不建議」 | 它回你：「你這條『暫緩擴張』是 2026-02 定的、註明每季複檢，現在要先更新嗎？」 |

差別不在 AI 聰不聰明，在你有沒有讓每一條記憶「知道自己會不會變」。

## AI 記憶其實有四層，每一層的信任方式都不同

把記憶攤開，它其實是四種完全不同的東西。混在一起就出事；分開放，它才知道哪一層可以閉著眼睛信、哪一層必須定期回頭查。

這不是紙上談兵——我自己的 AI 記憶就是用前綴分層存的（`事實_`、`判斷_`、`偏好_`、`參照_` 各一類檔）。下面每一層的寫法，都是我實際在用的。

### 第一層 事實：幾乎不變，但錯了最久

組織架構、各事業的基本背景、長期不變的規則、關鍵分工——這些幾乎不動，AI 可以一直依賴，不必每次重問你。但它最危險的地方，正是「AI 從不懷疑這裡」：你說什麼就是什麼，沒有第二個版本。一旦這層有一條錯了，它會錯得最久、最難被你發現。

拿去用——事實層我這樣寫（每季抽查一次有沒有過時）：

```
# 事實層（穩定，每季抽查）
- 我同時經營的事業：A（定位一句話）、B（定位一句話）
- 各事業的關鍵角色與分工
- 長期原則：決策先看現金流，再看成長
```

### 第二層 判斷：會過期，每條都要標日期

「目前策略是這個」「這件事先擱著」「這個人現階段適合做這個」——這些今天成立，下個月可能就不成立了。判斷有保存期限，必須標上「什麼時候定的」，並且定期回頭校正。我開頭那場差點出事的災難，就出在這一層被當成事實存了。

拿去用——判斷層我這樣寫，給每一條決定一個「保存期限」：

```
# 判斷層（會過期，標日期）
- 【判斷】A 暫緩擴張　【定於】2026-02　【依據】現金流吃緊　【複檢】每季
- 【判斷】B 暫不擴編　【定於】2026-04　【依據】淡季人力夠　【複檢】Q3 重評
```

就是補上 `【定於】` 和 `【複檢】` 這個小動作，成了我開頭那場災難的疫苗。但這裡要老實講清楚：**光標日期，只幫「你」在看到時察覺**；要讓 AI **主動**把日期報出來、提醒你更新，還得在偏好層加一條規則叫它這麼做——下一層就補上。

### 第三層 偏好：它開始懂你怎麼工作（最值錢、複製不走）

這一層存的不是事情，是「你這個人怎麼運作」：你怎麼下決定、什麼會讓你卡住、你喜歡結論先講還是推導先講、你討厭哪種格式。為什麼這層最值錢？因為事實 Google 也查得到、判斷換個工具也能重推，但「它記住你怎麼工作」是別人複製不走的。**你存進這一層的，其實不是資訊，是一段工作關係。**

拿去用——偏好層我這樣寫（這層最該慢慢養）：

```
# 偏好層（怎麼跟我工作）
- 給建議先說結論，再講推導
- 重大決策一定先列對立面與風險，再讓我拍板
- 不要模稜兩可的「看情況」；要給判斷＋你的假設
- 數字一律標假設來源
- 用到判斷層的條目，先報它的【定於】日期，再問現在還適不適用
```

最後那條，正是讓判斷層的日期「活起來」的開關：**判斷層負責記下日期，偏好層負責叫它每次都看**。兩層合用，AI 才會主動幫你擋掉過期判斷——這也是我開頭那場災難真正的解法，缺一層都不夠。

這層建到一個程度，你會發現它不再只是幫你做事，而是開始懂你怎麼做事——這層的完整建法，我留到下一篇講。

### 第四層 參照：不存內容，只存「去哪找」

最後一層很特別：它不存內容，只存指標。「這件事的細節在那份文件」「這個專案的脈絡去那裡看」。它很輕，卻很關鍵——讓系統知道自己的邊界：什麼不需要硬記，因為知道去哪查。

拿去用——參照層我這樣寫：

```
# 參照層（只存去哪找）
- A 財務細節 → 財務試算表
- 人事制度脈絡 → 員工手冊
- 品牌定位完整版 → 品牌總綱文件
```

一個成熟的記憶系統，不是什麼都記得，而是**知道自己不知道什麼、又知道去哪問**。

## 分層的重點：不是整齊，是知道「什麼可以信」

如果只能記一句，記這句：記憶分層不是為了整齊，是為了讓 AI 知道每一條能信到什麼程度。事實層閉著眼睛信，但每隔一陣子要抽查（錯了最久）；判斷層信，但要看日期；偏好層越厚越好，因為那是你的護城河；參照層不必信內容，信它指的方向。

當每一條記憶都「知道自己是哪一層」，那個「它用過期版本回答你」的災難，就從看不見變成擋得住。

## 今天就能做：三步，加一張可抄的骨架

不必一次重建，先做這三步：

1. **把現有記憶倒出來，每條問一句「這會變嗎？」** 不會變的歸事實，會變的歸判斷。光這個動作，你就會嚇到有多少「會過期的判斷」被你當成事實存著。
2. **給判斷層每一條補上 `【定於】`、`【複檢】`。** 讓時間風險浮上檯面。
3. **開一個專門放偏好的地方。** 每次你又得跟它解釋一次自己的工作習慣，就把那句話收進去。越早開始越值錢。

可以直接複製、改成你自己的記憶骨架：

```
# 我的 AI 記憶（分層）

## 事實（穩定，每季抽查）
- …

## 判斷（會過期，標日期）
- 【判斷】…　【定於】…　【依據】…　【複檢】…

## 偏好（怎麼跟我工作）
- …

## 參照（只存去哪找）
- … → …
```

## 常見問題 FAQ

**Q：AI 內建的記憶功能，能分層嗎？**
A：多數內建記憶不會主動幫你分層——它一樣是「塞進同一個地方」。分層是你給它的結構，不是它預設的功能。你可以用上面那種標題分段，自己把四層切出來。補充一點：有些工具的記憶是「自動管理」的，有些是你「手動維護」的檔——這篇四層講的是後者，也就是你自己寫、自己改的那份記憶檔。那是你最控制得住、也最該分層的部分。

**Q：這套記憶要用什麼工具存？**
A：一個純文字檔就能開始（`.txt` 或 `.md`），重點是「分層」這個習慣，不是工具。能版本控制（例如放進 git）會更徹底、看得到歷史，但先從一份文字檔貼給 AI 讀，就已經有效。

**Q：判斷層多久校正一次？**
A：看你的決策變動速度。我自己是每週掃一次（這正是上一篇講的矛盾校正）。重點是有固定節奏，不要等出錯才回頭。

**Q：一定要四層都做嗎？**
A：不用一次到位。最該先分的是「事實 vs 判斷」——光是把會過期的東西標出來，就能擋掉大半的記錯。偏好層慢慢養，參照層等內容多了再加。

**Q：我不是工程師，也能分層嗎？**
A：可以。分層不是技術，是分類習慣——問每條記憶「這會不會變」，就是分層的全部核心。

四層裡，最值錢也最難建的是偏好層——它記住的不是你做了什麼，是你怎麼做事。把這層建厚，AI 才會從「一個聰明的工具」變成「一個懂你的工作夥伴」。

下一篇我會專講這層怎麼建：讓它記住「怎麼跟你工作」——這才是最值錢的記憶（即將上線）。
